Цикл жизни приложения можно условно разделить на три этапа:  
- разработка :: Работа непосредственно с кодом. Ошибки, которые  
  выявлены на этом этапе, могут быть сразу исправлены.  
- тестирование :: Без прямого доступа к коду, но с возможностью  
  сопоставить с кодом. Ошибки, которые выявлены на этом этапе, можно  
  вернуть в разработку и там исправить.  
- эксплуатация :: Реальное использование приложения в рабочих  
  условиях. Ошибки имеют ощутимую цену, т.к. приводят к негативным  
  последствиям. Исследование ошибок затруднено.  
  
Вывод: Чем раньше удается обнаружить ошибку, тем ниже цена этой  
ошибки. В идеале ошибка обнаруживается сразу, как только она была  
допущена.  
  
Если ошибка обнаружена на этапе компиляции (compile time), то это  
более предпочтительно, чем если она обнаружена в процессе исполнения  
(runtime). Следовательно, предпочтительно иметь максимально строгие  
проверки в compile time.  
  
Программирование это процесс написания кода, который будет выполнять  
полезную функцию в runtime. Метапрограммирование это написание кода,  
который будет выполняться в compile time, т.е. различные compile-time  
проверки и кодогенерация.  
  
Некоторые языки имеют встроенные функции для метапрограммирования. В  
этом случае инструкции для генерации кода пишутся на том же языке, что  
и остальная программа.  
  
В C++ для генерации кода используются макросы препроцессора и шаблоны.  
Кроме того, можно использовать внешние инструменты (autoconf/automake,  
cmake configure_file или отдельные скрипты) для генерации кода на  
этапе сборки.  
  
В C++ поддерживаются compile-time вычисления, которые могут быть  
реализованы либо через рекурсивные шаблоны, либо через автоматическое  
вычисление параметров шаблонов. Кроме того, базовая арифметика может  
быть также выполнена на этапе компиляции средствами компилятора.  
  
Пример compile-time вычисления через вычисление параметров шаблона:  
#+begin_src c++  
  template <size_t N>  
  constexpr size_t cstrlen(const char (&)[N]) noexcept {  
    return N - 1;  
  }  
  
  // cstrlen("Hello, world!") заменяется на 13 в процессе компиляции  
#+end_src  
  
- Замечание :: Компилятор не всегда способен вычислить параметры  
  шаблона, только для некоторых заранее описанных случаев. В остальных  
  случаях будет выдаваться сообщение об ошибке.  
- правила вычисления параметров шаблона :: [https://en.cppreference.com/w/cpp/language/template_argument_deduction.html](https://en.cppreference.com/w/cpp/language/template_argument_deduction.html)  
  
Пример compile-time вычисления с помощью рекурсивного шаблона:  
#+begin_src c++  
  template <int N>  
  struct Fib {  
    static constexpr int value = Fib<N-1>::value + Fib<N-2>::value;  
  };  
  
  template<> struct Fib<0> {  
    static constexpr int value = 0;  
  };  
  
  template<> struct Fib<1> {  
    static constexpr int value = 1;  
  };  
  
  // Fib<10>::value заменяется на 55 в процессе компиляции  
#+end_src  
  
Если функция или класс принимает некоторый параметр, можно вместо  
этого сделать этот параметром шаблонным параметром. Это позволяет  
реализовать compile-time специализацию функций.  
  
#+begin_src c++  
  void message(const char *user, bool greeting) {  
    if (greeting) // runtime ветвление  
      printf("Hello, %s\n", user);  
    else  
      printf("Goodbye, %s\n", user);  
  }  
  
  template<bool greeting>  
  void message2(const char *user) {  
    if (greeting) // compile-time ветвление  
      printf("Hello, %s\n", user);  
    else  
      printf("Goodbye, %s\n", user);  
  }  
#+end_src  
  
- Замечание :: Более надежно использовать constexpr if.  
  
#+begin_src c++  
  template<bool greeting>  
  void message2(const char *user) {  
    if constexpr (greeting) // гарантированное compile-time ветвление  
      printf("Hello, %s\n", user);  
    else  
      printf("Goodbye, %s\n", user);  
  }  
#+end_src  
  
При упоминании параметризованного шаблона в коде происходит  
инстанцирование шаблона. При этом происходит генерация шаблонного кода  
с конкретным значением шаблонных параметров. Компилятор может не  
дублировать части кода, которые совпадают. Кроме того, каждый  
экземпляр шаблона является полноценным символом и его части могут быть  
переопределены (см пример с числами Фибоначчи, где =Fib<0>= и =Fib<1>=  
переопределены относительно базового случая).  
  
Кроме обычных значений параметром шаблона также может быть тип.  
#+begin_src c++  
  template <typename T>  
  struct MyWrapper {  
    T value;  
  };  
  
  MyWrapper<int> x = { 10 };  
  MyWrapper<const char *> y = { "Hello, world" };  
#+end_src  
  
Шаблон может иметь несколько параметров или переменное число  
параметров (вариадические шаблоны). Существует также возможность  
использовать в шаблонах логические условия, описывающие те или иные  
свойства типов, используемых в качестве параметра. Если при попытке  
вычисления параметров шаблона обнаруживается, что условия не  
выполняются, компилятор не считает это за ошибку, а пытается применить  
другой шаблон. Данное поведение называется принципом SFINAE  
(Substitution Failure Is Not An Error).
